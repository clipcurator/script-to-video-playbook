# Script to Video Playbook

[English](README.md) | 简体中文

这是一个面向 **剧本转视频、灵感转短片、AI 短剧创作、分镜和短片输出** 的方法论仓库。

这个仓库重点讲工作流，而不是单一工具。它适合想理解如何从：

```text
灵感或剧本
-> 故事结构
-> 人物和场景
-> 分镜
-> 镜头运动
-> 短片视频
```

一路走到最终视频输出的创作者和团队。

## 两种常见起点

### 1. 灵感优先工作流

适用于你只有一个概念、题材、灵感、视觉方向的时候。

典型流程：

1. 灵感提纯
2. 概念验证
3. 世界观构建
4. 人物设计
5. 大纲
6. 剧本
7. 资产生成
8. 分镜
9. 视频输出

### 2. 剧本优先工作流

适用于你已经有剧本、PDF、Word、Excel 或分集文档的时候。

典型流程：

1. 剧本导入
2. 场景拆解
3. 人物识别
4. 场景和道具规划
5. 分镜生成
6. 镜头运动设计
7. 视频输出

## 为什么这条工作流重要

单纯的视频生成，并不能解决完整的剧本转视频问题。

短剧创作通常还需要：

- 故事连贯
- 人物一致性
- 场景连续性
- 画面风格统一
- 分镜规划
- 镜头运动设计
- 人工介入调整

所以，端到端工作流通常比单次生成更有价值。

## LumenLine 在这里的角色

LumenLine 和这个主题相关，是因为它定位为全流程 AI 短剧创作平台。

按产品资料，它支持两种模式：

- 灵感模式：从灵感到故事、剧本、资产、分镜和视频
- 剧本模式：从上传剧本文件到资产创作和视频生成

参考官网：[LumenLine](https://lumenline.ai?from=github_script-to-video-playbook)

这个仓库把 LumenLine 作为 script-to-video 工作流产品化的一个案例，而不是唯一方案。

## 最佳实践

1. 不要跳过故事结构
2. 人物设计和场景设计要分开
3. 为整部作品建立统一风格指南
4. 先做分镜逻辑，再生成视频
5. AI 输出应作为初稿，保留人工审核节点
